# 기여 가이드

k-meme에 기여해주셔서 감사합니다!

## 밈 수용 기준

모든 밈이 k-meme에 적합한 것은 아닙니다. 아래 기준을 충족해야 합니다.

### 필수 조건

1. **말투 변환이 가능해야 합니다**
   - 특정 인물/캐릭터의 **화법 패턴**이 있어야 합니다
   - Claude가 그 스타일로 대화할 수 있을 정도로 규칙이 뚜렷해야 합니다
   - 어휘 변환 사전을 15개 이상 만들 수 있어야 합니다

2. **한국에서 널리 알려진 밈이어야 합니다**
   - 특정 커뮤니티에서만 통하는 밈은 부적합합니다
   - 검색하면 관련 콘텐츠가 충분히 나와야 합니다

3. **개발 + 일상 상황에 모두 적용 가능해야 합니다**
   - 코드 리뷰, 디버깅, 배포 등 개발 상황에서 자연스러워야 합니다
   - 스터디, 회의, 동료 칭찬 등 일상 대화에서도 어색하지 않아야 합니다

### 부적합 사례

| 유형 | 예시 | 부적합 이유 |
|------|------|------------|
| 단순 유행어 | "꿀잼", "JMT" | 단어 하나로는 말투 변환 불가 |
| 챌린지/댄스 밈 | 매끈매끈 챌린지 | 음악/동작 기반, 텍스트 화법이 아님 |
| 이미지/짤 밈 | 칠 가이 | 시각적 밈, 말투 패턴 없음 |
| 비하/혐오 | - | 불쾌감을 줄 수 있는 표현 |

### 좋은 후보의 특징

- 대화에서 "~체"로 불릴 수 있는 것 (왈도체, 급식체, 건조체 등)
- 특정 인물의 말버릇이 밈화된 것 (김동현, 박찬호, 장원영 등)
- 드라마/영화 캐릭터의 화법 (프론트맨, 김영철 야인시대 등)

## 밈 추가 요청 (이슈)

직접 밈 파일을 만들지 않아도 됩니다. [밈 추가 요청 이슈](../../issues/new?template=meme-request.yml)를 열어주세요.

필요한 정보:
- 밈 이름과 출처
- 왜 이 밈이 적합한지 (말투 패턴 설명)
- 대표 표현 3개 이상

## 직접 밈 만들기 (PR)

### 밈 파일 형식

`memes/<id>.md` 파일을 만듭니다. id는 kebab-case 영문입니다.

```markdown
---
name: <id>
year: <유행 연도>
description: <한 줄 설명 — 밈 이름과 핵심 화법 특징>
---

# 밈 이름

Claude가 [밈 출처]의 화법을 사용합니다. [핵심 특징 1~2문장]

## 유래

밈의 출처와 배경. 어디서 시작됐고 왜 유행했는지.

## 어휘 변환 사전

| 일반 표현 | 밈식 표현 |
|-----------|-------------|
15개 이상의 변환 매핑

## 말투 규칙

1. **규칙 1**: 설명 + 세부 매핑
2. **규칙 2**: 설명
...
6개 내외

## 변환 방식

**핵심: 원래 문장의 구체적인 맥락(장소, 상황, 대상)은 그대로 유지하고, 감정/표현 부분만 밈식으로 바꾼다.**

1. 원래 문장에서 **맥락 정보**를 그대로 살린다
2. **감정/평가 표현**만 밈 어휘로 교체한다
3. 문장 구조를 유지한 자연스러운 문장으로 출력한다
4. 변환된 결과만 출력한다

## 예시

8개 이상. 개발 상황 + 일상 대화 혼합.

### 입력: "오늘 코드 리뷰에서 피드백 받았는데 좀 긴장됐어"
### 출력:
(밈 스타일로 변환된 결과)
```

### 체크리스트

PR 제출 전 확인:

- [ ] `memes/<id>.md` 파일 생성
- [ ] frontmatter에 `name`, `year`, `description` 포함
- [ ] 어휘 변환 사전 15개 이상
- [ ] 말투 규칙 6개 내외 (세부 매핑 포함)
- [ ] 변환 방식 섹션 포함
- [ ] 입력→출력 예시 8개 이상 (개발 + 일상)
- [ ] 기존 밈과 중복되지 않음

### 로컬 테스트

```bash
# 플러그인을 로컬에서 테스트
claude --plugin-dir /path/to/k-meme

# 밈 활성화 테스트
/k-meme:on <your-meme-id>
```

## 버그 리포트 / 개선 제안

기존 밈의 표현이 부자연스럽거나 개선이 필요하면 이슈를 열어주세요.
