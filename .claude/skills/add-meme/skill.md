# 밈 추가 오케스트레이터

새로운 한국 밈을 k-meme에 추가하는 3단계 파이프라인을 실행한다.

## 워크플로우

### Phase 1: 스카우트 (조사)

밈 스카우트 에이전트를 실행하여 밈 후보를 조사하고 적합성을 평가한다.

**입력이 구체적인 밈 이름인 경우** (예: "곽철용 밈 추가해줘"):
- 해당 밈의 상세 정보를 조사하고 적합성 평가

**입력이 모호한 경우** (예: "새 밈 추가해줘", "요즘 뜨는 밈"):
- 트렌드를 조사하여 후보 3~5개를 제시하고 사용자에게 선택 요청

**실행:**
```
Agent(
  name: "scout",
  agent: "meme-scout",
  model: "opus",
  prompt: "<밈 이름/키워드>에 대해 조사하고 k-meme 적합성을 평가해줘.
           기존 밈 목록: !`ls memes/*.md | xargs -I{} basename {} .md`"
)
```

**중단 조건:** 스카우트가 "부적합"으로 판단하면 사용자에게 이유를 설명하고 다른 밈을 제안한다.

### Phase 2: 작성

밈 작성자 에이전트를 실행하여 밈 정의 파일을 생성한다.

**실행:**
```
Agent(
  name: "writer",
  agent: "meme-writer",
  model: "opus",
  prompt: "다음 밈을 memes/<id>.md로 작성해줘.
           <Phase 1 스카우트 결과 요약>
           품질 기준은 memes/kim-dong-hyun.md를 참조해."
)
```

### Phase 3: 리뷰

밈 리뷰어 에이전트를 실행하여 생성된 파일의 품질을 검증하고 필요시 수정한다.

**실행:**
```
Agent(
  name: "reviewer",
  agent: "meme-reviewer",
  model: "opus",
  prompt: "memes/<id>.md 파일을 리뷰하고, 품질이 부족한 부분은 직접 수정해줘."
)
```

### Phase 4: 완료 보고

사용자에게 결과를 보고한다:
- 추가된 밈 이름과 ID
- `/k-meme:on <id>`로 바로 사용 가능하다는 안내
- 리뷰어의 품질 점수

## 에러 핸들링

| 상황 | 대응 |
|------|------|
| 스카우트가 부적합 판정 | 사용자에게 이유 설명, 대안 밈 제안 |
| 기존 밈과 중복 | 기존 밈 ID 안내, 개선 여부 질문 |
| 작성자가 표현 15개 미달 | 리뷰어가 웹 검색으로 보충 |
| 리뷰 점수 6/10 미만 | 작성자에게 재작성 요청 (1회) |

## 테스트 시나리오

### 정상 흐름
```
사용자: "곽철용 밈 추가해줘"
→ 스카우트: 곽철용 조사, 적합 판정
→ 작성자: memes/gwak-chul-yong.md 생성
→ 리뷰어: 검증 + 수정, 8/10
→ 완료: "/k-meme:on gwak-chul-yong 으로 사용하세요"
```

### 에러 흐름
```
사용자: "강남스타일 밈 추가해줘"
→ 스카우트: 조사 후 "부적합 — 노래/춤 밈이라 말투 변환 불가"
→ 사용자에게 안내: "대신 싸이의 인터뷰 화법은 어떨까요?"
```
