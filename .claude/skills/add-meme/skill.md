# 밈 추가 오케스트레이터

새로운 한국 밈을 k-meme에 추가한다. **생성-검증(Producer-Reviewer)** 패턴으로, 작성자와 리뷰어가 품질 기준을 충족할 때까지 반복한다 (최대 2회 재시도).

## 아키텍처

```
[스카우트] → [작성자] ⇄ [리뷰어] → 완료
              ↑    피드백 루프    ↓
              └── (품질 미달 시) ──┘
              최대 2회 재시도
```

## 워크플로우

### Phase 1: 스카우트 (조사)

밈 스카우트 에이전트를 실행하여 밈 후보를 조사하고 적합성을 평가한다.

**입력이 구체적인 밈 이름인 경우** (예: "곽철용 밈 추가해줘"):
- 해당 밈의 상세 정보를 조사하고 적합성 평가

**입력이 모호한 경우** (예: "새 밈 추가해줘", "요즘 뜨는 밈"):
- 트렌드를 조사하여 후보 3~5개를 제시하고 사용자에게 선택 요청

**실행:**
```
Agent(
  name: "scout",
  agent: "meme-scout",
  model: "opus",
  prompt: "<밈 이름/키워드>에 대해 조사하고 k-meme 적합성을 평가해줘.
           기존 밈 목록: !`ls memes/*.md | xargs -I{} basename {} .md`"
)
```

**중단 조건:** 스카우트가 "부적합"으로 판단하면 사용자에게 이유를 설명하고 다른 밈을 제안한다.

### Phase 2~3: 생성-검증 루프

작성자가 밈 파일을 생성하고, 리뷰어가 검증한다. 품질 미달 시 리뷰어의 피드백을 작성자에게 전달하여 재작성한다. **최대 2회 재시도** 후에도 미달이면 현재 상태로 저장하고 사용자에게 수동 개선을 안내한다.

#### 첫 번째 작성 (Phase 2)

```
Agent(
  name: "writer",
  agent: "meme-writer",
  model: "opus",
  prompt: "다음 밈을 memes/<id>.md로 작성해줘.
           <Phase 1 스카우트 결과 요약>
           품질 기준은 memes/kim-dong-hyun.md를 참조해."
)
```

#### 검증 (Phase 3)

```
Agent(
  name: "reviewer",
  agent: "meme-reviewer",
  model: "opus",
  prompt: "memes/<id>.md 파일을 리뷰해줘. 아래 기준으로 점수를 매기고,
           품질이 부족한 부분은 구체적인 피드백을 줘.

           검증 기준:
           - 어휘 변환 사전 15개 이상
           - 말투 규칙 6개 + 세부 매핑
           - 변환 방식 4단계
           - 입력→출력 예시 8개+ (개발+일상 혼합)
           - 어휘 사전 표현이 예시에서 실제 사용됨
           - 변환된 문장이 해당 인물이 할 법한 말인지

           출력 형식:
           - 점수: X/10
           - 통과 여부: PASS (7점 이상) / FAIL
           - FAIL 시 구체적 피드백: 무엇이 부족하고, 어떻게 고쳐야 하는지"
)
```

#### 피드백 루프 (FAIL 시)

리뷰어가 FAIL 판정을 내리면 피드백을 작성자에게 전달하여 재작성:

```
Agent(
  name: "writer-retry",
  agent: "meme-writer",
  model: "opus",
  prompt: "memes/<id>.md를 리뷰어 피드백에 따라 수정해줘.

           리뷰어 피드백:
           <Phase 3 리뷰어의 구체적 피드백>

           수정 후 다시 품질 기준을 자체 검증해:
           - 어휘 사전 15개+, 예시 8개+, 사전 표현이 예시에서 사용되는지"
)
```

재작성 후 다시 Phase 3 검증을 실행한다. 이 루프는 **최대 2회**까지만 반복한다.

### Phase 4: 완료 보고

사용자에게 결과를 보고한다:
- 추가된 밈 이름과 ID
- 리뷰어의 최종 점수
- 루프 횟수 (1회 통과 / 재시도 N회 후 통과)
- `/k-meme:on <id>`로 바로 사용 가능하다는 안내

## 에러 핸들링

| 상황 | 대응 |
|------|------|
| 스카우트가 부적합 판정 | 사용자에게 이유 설명, 대안 밈 제안 |
| 기존 밈과 중복 | 기존 밈 ID 안내, 개선 여부 질문 |
| 리뷰 FAIL (1~2회차) | 피드백을 작성자에게 전달, 재작성 |
| 리뷰 FAIL (3회차, 최종) | 현재 상태로 저장, 사용자에게 수동 개선 안내 |

## 테스트 시나리오

### 정상 흐름 (1회 통과)
```
사용자: "곽철용 밈 추가해줘"
→ 스카우트: 조사, 적합 판정
→ 작성자: memes/gwak-chul-yong.md 생성
→ 리뷰어: 8/10, PASS
→ 완료: "/k-meme:on gwak-chul-yong 으로 사용하세요"
```

### 피드백 루프 흐름
```
사용자: "나영석 밈 추가해줘"
→ 스카우트: 조사, 적합 판정
→ 작성자: memes/na-young-seok.md 생성
→ 리뷰어: 5/10, FAIL — "예시가 개발 상황에 편중, 일상 예시 부족"
→ 작성자 재시도: 피드백 반영하여 수정
→ 리뷰어: 8/10, PASS
→ 완료: "1회 재시도 후 통과. /k-meme:on na-young-seok"
```

### 부적합 흐름
```
사용자: "강남스타일 밈 추가해줘"
→ 스카우트: "부적합 — 노래/춤 밈이라 말투 변환 불가"
→ 사용자에게 안내: "대신 싸이의 인터뷰 화법은 어떨까요?"
```
